⚙️ Comandos Esenciales de Git

Git es un sistema de control de versiones. Imagina que es como un "viaje en el tiempo" o un sistema de autoguardado súper avanzado para tus proyectos de código.

1. Iniciar un proyecto: git init

Convierte una carpeta normal de tu computadora en un repositorio de Git local.

git init

2. Ver el estado: git status

Es el comando que más usarás. Te dice en qué rama estás, qué archivos has modificado, qué archivos nuevos hay, y cuáles están listos para guardarse.

git status

3. Guardar cambios: git commit

Guardar en Git requiere dos pasos: primero "preparas" los archivos (add) y luego creas el "punto de guardado" (commit) con un mensaje descriptivo.

# 1. Preparar todos los cambios
git add .

# 2. Crear el commit
git commit -m "Mi primer mensaje de guardado"

4. Cambio de rama: git branch y git checkout

Las ramas (branches) te permiten experimentar sin romper el código principal. Puedes crear una rama nueva o cambiar a una existente.

# Ver todas las ramas
git branch

# Crear y cambiar a una rama nueva
git checkout -b mi-nueva-funcion
# (o en versiones nuevas de Git: git switch -c mi-nueva-funcion)

# Cambiar de vuelta a la rama principal (main)
git checkout main
# (o git switch main)

🌊 Entendiendo Gitflow

Gitflow es un modelo de trabajo (una forma de organizarse) para usar las ramas de Git en proyectos más grandes. Define un conjunto estricto de ramas principales y de soporte:

  • main (o master): El código perfecto y en producción.
  • develop: Donde integramos todas las nuevas funciones antes de lanzarlas.
  • feature/...: Ramas temporales donde programamos cada nueva funcionalidad.

🤔 Recomendaciones de Flujos de Trabajo

Escenario¿Qué flujo usar?Descripción
Para aprender o proyectos personales pequeños Trunk-based básico (Solo Main) Trabaja directamente en main. Haz add y commit frecuentemente. Es perfecto para empezar y no complicarse con ramas hasta que domines lo básico.
Trabajar solo (nivel intermedio) GitHub Flow (Main + Feature branches) La rama main siempre está limpia. Si vas a agregar algo nuevo, creas una rama feature/mi-idea, haces tus cambios, y luego la unes (merge) a main.
Trabajar en equipo completo GitHub Flow con Pull Requests (Recomendado hoy) o Gitflow Clásico En equipos modernos, GitHub Flow es más ágil. Cada developer crea una rama, sube su código y abre un "Pull Request" para que el equipo revise el código antes de unirlo a main. Gitflow Clásico se usa en proyectos corporativos con ciclos de lanzamiento estáticos y complejos.
🌟 En este curso: Te recomendamos empezar usando solo main para tus primeros ejercicios si trabajas solo. ¡Concéntrate en entender cómo funcionan status, add y commit primero!
