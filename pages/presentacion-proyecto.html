Guía de Proyecto y Especificación Técnica

Resumen del Proyecto

El objetivo de este proyecto es que los estudiantes profundicen en conceptos avanzados de Python aplicados a la lógica de programas de terminal (CLI) y traduzcan ese conocimiento técnico en un recurso educativo digital (página web).

# Habilidades a desarrollar:
- Programación robusta
- Estructuración de datos complejas
- Comunicación técnica (Docs)

A continuación se presentan los 3 Pilares de Investigación. Cada estudiante deberá elegir un módulo de forma individual para investigar y desarrollar.

Módulo 1 Algoritmia

Algoritmia y Ordenamiento en Consola

Fundamento:
Lógica de comparación y ordenamiento detrás de las funciones nativas.

Requerimiento Técnico:
Comparar el uso de .sort() vs sorted().

Implementar una versión simplificada de Bubble Sort o Merge Sort que imprima en la terminal cada paso del proceso (visualización por pasos).

> Iniciando Bubble Sort...
Paso 1: [3, 1, 4, 2] -> Intercambiando 3 y 1
Paso 2: [1, 3, 4, 2] -> 3 y 4 en orden
Paso 3: [1, 3, 4, 2] -> Intercambiando 4 y 2
Resultado parcial: [1, 3, 2, 4]
...
KPI de Aprendizaje
Entender la complejidad básica y la diferencia entre métodos in-place y aquellos que retornan nuevas listas.

Módulo 2 Manejo de Errores

Gestión de Excepciones

Fundamento:
Robustez de software y experiencia de usuario en terminal.

Requerimiento Técnico:
Crear un programa que solicite múltiples inputs y gestione mediante bloques try/except/finally al menos 3 tipos de errores comunes:

  • ValueError
  • FileNotFoundError
  • ZeroDivisionError
try:
    edad = int(input("Edad: "))
except ValueError:
    print("❌ Error: Ingresa un número entero.")
KPI de Aprendizaje
Evitar el "crash" o caída abrupta del programa y proveer feedback útil al usuario.

Módulo 3 F-Strings

F-Strings y Reportes Profesionales

Fundamento:
Formateo avanzado de strings y presentación de datos.

Requerimiento Técnico:
Uso de f-strings para: redondeo de decimales, alineación (izquierda, derecha, centro) y relleno de caracteres.

El entregable debe ser un generador de "Tickets" o "Reportes" perfectamente tabulados.

==================================
        TICKET DE COMPRA        
==================================
Item           Cant     Precio
----------------------------------
Teclado Mec...   2      $45.99
Monitor 24"...   1     $120.50
==================================
TOTAL:                 $212.48
KPI de Aprendizaje
Dominar el control total sobre la salida de texto en consola sin depender de librerías externas.

Especificaciones del Entregable Web

La página web no es solo un adorno; es el Manual Técnico del tema elegido.

🎨 Estética Sugerida

Basada en terminales (Dark mode, fuentes monoespaciadas tipo JetBrains Mono, Fira Code o Courier).

🖱️ Interactividad

Debe incluir bloques de código resaltados y, de ser posible, una simulación visual (vía CSS o texto) de cómo se ve el programa ejecutándose en la terminal.

📑 Estructura Requerida

  • Título y Definición del concepto.
  • Sección "Antes vs. Después" (Código ineficiente vs. Código optimizado).
  • Diagrama de flujo o infografía técnica.
  • Snippet de código interactivo.

Guía: Cómo subir tu proyecto a la Web

Puedes usar la interfaz de VS Code, o mejor aún, demostrar tus habilidades usando la terminal integrada:

1. Prepara tu Repositorio

Crea un repositorio vacío en GitHub. Luego abre tu terminal en la carpeta del proyecto e inicializa Git:

$ git init

2. Prepara tus Archivos

Añade tus archivos y realiza tu primer guardado (commit):

$ git add .
$ git commit -m "Versión inicial"

3. Conecta y Sube a GitHub

Vincula tu código local con GitHub y sube tu proyecto:

$ git remote add origin <URL-GITHUB>
$ git branch -M main
$ git push -u origin main

4. ¡Activa la Web! 🚀

En tu repositorio de GitHub, ve a Settings > Pages. En "Branch", selecciona "main" y guarda. ¡Tu link estará listo en breve!
