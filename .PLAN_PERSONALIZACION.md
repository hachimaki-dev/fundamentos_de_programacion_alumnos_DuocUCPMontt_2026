# 📋 PLAN: Full Personalización de Perfiles — Academia hachimaki

## 🎯 Objetivo General
Crear un sistema de personalización completo donde los usuarios puedan editar su perfil, subir fotos, y acceder al perfil de otros usuarios desde una experiencia integrada única. El perfil y MySpace se unifican en una sola entidad.

---

## 📐 Arquitectura Propuesta

### 1️⃣ **Estructura de Datos** (Base de Datos)

**Cambios en `profiles` table:**
```sql
-- Campos nuevos a agregar:
- avatar_custom_url TEXT        -- URL de foto custom subida
- avatar_source TEXT DEFAULT 'catalog'  -- 'catalog' | 'custom'
- bio TEXT                       -- Descripción corta del perfil
- custom_html TEXT               -- MySpace HTML (existe)
- custom_css TEXT                -- MySpace CSS (existe)
- social_links JSONB             -- Links sociales: {twitter, github, youtube, etc}
- visibility TEXT DEFAULT 'public'  -- 'public' | 'private' | 'followers_only'
- updated_at TIMESTAMPTZ         -- (ya existe, pero importante)
- myspace_theme TEXT DEFAULT 'default'  -- Para futuras plantillas
```

**Nueva tabla: `profile_followers` (relación M-to-M)**
```sql
- id UUID PRIMARY KEY
- follower_id UUID REFERENCES profiles(id)
- following_id UUID REFERENCES profiles(id)
- created_at TIMESTAMPTZ DEFAULT now()
- UNIQUE(follower_id, following_id)
```

**Nueva tabla: `profile_views` (analytics)**
```sql
- id UUID PRIMARY KEY
- profile_id UUID REFERENCES profiles(id)
- viewer_id UUID REFERENCES profiles(id) (nullable, for anon)
- viewed_at TIMESTAMPTZ DEFAULT now()
```

### 2️⃣ **Buckets de Storage**

**Crear bucket: `profile-avatars`**
```
- Privado (requiere auth para acceso)
- Límite: 5MB por imagen
- Tipos MIME: image/jpeg, image/png, image/webp, image/gif
- Estructura: /profile-avatars/{user_id}/{filename}
```

## 🔧 Funcionalidades Técnicas

### 1. **Upload de Avatar**
```javascript
// Workflow:
1. Usuario selecciona imagen
2. Validar en cliente (tipo, tamaño)
3. Comprimir si es necesario (opcional)
4. Upload a Supabase Storage
5. Actualizar campo avatar_custom_url en profiles
6. Cambiar avatar_source a 'custom'
7. Preview actualizado
```

**Seguridad:**
- Validar tipos MIME en servidor
- Límite de tamaño (5MB)
- Sanitizar nombres de archivo
- Usar signed URLs con expiración para acceso

### 2. **Edición de Perfil**
```javascript
// Estado del formulario:
- Loading: mientras se guarda
- Success: confirmación (toast)
- Error: mostrar mensaje
- Dirty checking: avisar si hay cambios sin guardar

// Validaciones:
- Nickname: min 3 chars, alphanumeric + underscore, unique (async)
- Bio: max 160 caracteres
- URL de portfolio: validar URL format
- Dates: no futuro
```

### 3. **RLS (Row Level Security)**
```sql
-- Profiles
- SELECT: usuarios logueados pueden ver profiles públicos
- SELECT: usuarios ven su propio profile completo
- UPDATE: usuarios solo actualizan su propio profile

-- Profile Followers
- SELECT: todos pueden ver (para stats públicas)
- INSERT: usuarios autenticados
- DELETE: solo creador de la relación

-- Profile Views (si aplica metadata)
- INSERT: automático cuando se visita
- SELECT: restringido
```

## 📊 Flujos de Usuario

### Flujo 1: **Mi Primer Perfil Personalizado**
```
1. Usuario entra a Dashboard
2. Ve profile card incompleta
3. Hace click en "Editar Perfil"
4. Sube foto
5. Completa información básica (nombre, bio)
6. Guarda
7. Ve preview actualizado en Dashboard
```

### Flujo 2: **Visitar Perfil de Otro Usuario**
```
1. Desde feed, competencias, o sugerencias → hace click en avatar/nombre
2. Navega a profile.html?nickname=usuario
3. Ve profile público (si visibility = public)
4. Si es private: "Este perfil es privado"
5. Si es followers_only: "Síguelo para ver su perfil"
6. Botón de seguir/unfollow
```

### Flujo 3: **Personalizar MySpace**
```
1. En profile editor: sección "Personalización Avanzada"
2. Opción: "Editar HTML/CSS"
3. Abre modal con código
4. Permite HTML/CSS (sanitizado)
5. Preview en tiempo real
6. Guarda cambios
```
