🍝 El Problema: El Código Espagueti

Hasta ahora, has escrito scripts lineales. Todo el código va de arriba hacia abajo. Si necesitas hacer lo mismo tres veces (como validar una entrada o calcular un impuesto), copias y pegas el código. Esto genera archivos gigantescos, difíciles de leer y una pesadilla para encontrar errores.

La solución: "Empaquetar la lógica en bloques independientes que tengan un nombre y puedan ser llamados cuando los necesite".

🛠️ La Función: Tu Propia Herramienta

Una función es un bloque de código reutilizable diseñado para hacer una única tarea. Funciona como una máquina en una fábrica: le pasas materias primas (argumentosLos valores reales que le envías a la función cuando la llamas. Ej: sumar(5, 10).), hace su trabajo internamente y te devuelve el producto terminado (returnInstrucción que finaliza la función y devuelve un valor específico al código que la llamó.).

💡 La Regla de Oro: DRY (Don't Repeat Yourself)

Si te sorprendes copiando y pegando el mismo bloque de código más de dos veces en tu archivo, ¡es el momento exacto de crear una función!

📄 Anatomía de una Función datos ➡️ función ➡️ resultado 
# 1. DEFINICIÓN: Creamos la herramienta usando 'def'
def saludar(nombre):  # 'nombre' es el PARÁMETRO
    mensaje = f"¡Hola, {nombre}!"
    return mensaje    # Devolvemos el resultado al mundo exterior

# 2. LLAMADA: Usamos la herramienta
# Le pasamos "Ash" como ARGUMENTO y guardamos lo que retorna
saludo_ash = saludar("Ash") 
print(saludo_ash)  # ¡Hola, Ash!

print(saludar("Misty"))  # ¡Hola, Misty!

📥 Entradas: ParámetrosVariable utilizada en la definición de la función. y ArgumentosEl valor real que le pasas.

Las funciones necesitan datos para trabajar. Hay que distinguir entre dos conceptos clave que suelen confundirse:

📦 Parámetro (La caja)

Es la variable que pones dentro de los paréntesis al definir la función. Es un marcador de posición, como una caja vacía esperando contenido.

def sumar(a, b): # a y b son parámetros

🍎 Argumento (El contenido)

Es el valor real y concreto que le envías a la función al llamarla. Es lo que metes dentro de la caja para que la función lo procese.

sumar(5, 10) # 5 y 10 son los argumentos

Valores por Defecto: Puedes asignar un valor opcional a un parámetro. Si quien llama a la función no envía ese dato, se usará el valor por defecto.

def presentar(nombre, rol="estudiante"):
    print(f"{nombre} es {rol}")

presentar("Ash") # Usa el por defecto: Ash es estudiante
presentar("Oak", "profesor") # Sobrescribe: Oak es profesor

✨ Tip Pro: Type Hinting

En el código moderno, los desarrolladores usan Type Hinting (Pistas de Tipo). Aunque Python no te obliga, es una excelente práctica decir de qué tipo son los parámetros y qué tipo devuelve la función. ¡Tu IDE (y tus compañeros de trabajo) te lo agradecerán!

# Indica que 'a' y 'b' deben ser enteros (int)
# y que la función devolverá (->) un entero (int)
def sumar(a: int, b: int) -> int:
    return a + b

def saludar(nombre: str) -> str:
    return f"Hola {nombre}"

Veremos esto en mucha más profundidad más adelante, ¡pero es bueno que lo vayas conociendo!

🔒 El ScopeEl alcance de una variable. Define dónde existe y dónde puede ser leída.: Lo que pasa en la función, se queda en la función

Las variables creadas dentro de una función son locales. Solo existen mientras la función está trabajando. Cuando la función termina, esas variables se destruyen.

⚠️ Cuidado con intentar acceder al interior

def calcular_area(base, altura):
    area = (base * altura) / 2
    return area

calcular_area(10, 5)
print(area) # ❌ NameError: name 'area' is not defined

La variable area murió al terminar la función. Para usar su valor, debes capturar el return: mi_area = calcular_area(10, 5) y luego print(mi_area).

📤 Salidas: return vs print

Este es el concepto más malentendido por los principiantes.

  • 🖨️ print() solo es un "megáfono". Muestra cosas en la pantalla para que el humano las lea, pero al programa se le olvidan inmediatamente. No puedes guardar un print en una variable.
  • 📬 return es un "paquete de envío". Saca la información de la función y se la entrega al resto del programa para que pueda usarla en otros cálculos.

⚡ El superpoder del Return

El return tiene un poder extra: detiene la función inmediatamente. Es la puerta de salida. Cualquier línea de código escrita debajo de un return ejecutado, será ignorada por Python (Dead Code).

¡También puedes devolver múltiples cosas a la vez!

def estadisticas(notas):
    # Retorna tres valores separados por coma (Python los empaqueta en una tupla)
    return max(notas), min(notas), sum(notas)/len(notas)

# Desempaquetado mágico en 3 variables:
mayor, menor, prom = estadisticas([5.0, 7.0, 6.0])

Errores Cuáticos (FAQ)

Cosas que los estudiantes suelen hacer pensando que van a funcionar... pero desatan el caos absoluto.

1. El Return Olvidado (El fantasma del NoneType) ▼

❌ Lo que intentas:

def sumar(a, b):
    resultado = a + b
    # (¡Se te olvidó el return!)

mi_suma = sumar(5, 5)
print(mi_suma + 10)  # TypeError: unsupported operand type(s) for +: 'NoneType' and 'int'

✅ Lo que pasó:

Si una función no tiene un return explícito, Python asume que quieres retornar None (Nada). Entonces, mi_suma vale None. Cuando intentas sumar None + 10, el programa colapsa. ¡Asegúrate siempre de retornar lo que calculas!

2. Llamar sin Paréntesis (Imprimiendo la Matrix) ▼

❌ Lo que intentas:

def saludar():
    return "Hola a todos"

mensaje = saludar
print(mensaje)  # <function saludar at 0x7f8b9c...>

✅ Lo que pasó:

Si escribes el nombre de la función SIN paréntesis, NO la estás llamando, solo estás apuntando al lugar en la memoria (RAM) donde está guardado su código. Para que la función "trabaje", debes usar los paréntesis: saludar().

3. La Trampa Mutante (Listas como Default) ▼

❌ Lo que intentas:

def agregar_item(item, inventario=[]):
    inventario.append(item)
    return inventario

print(agregar_item("Espada"))  # ['Espada']
print(agregar_item("Escudo"))  # ['Espada', 'Escudo']  ¡¿Qué pasó?!

✅ Lo que pasó:

NUNCA uses una lista o diccionario vacío `[]` / `{}` como valor por defecto de un parámetro. Python crea la lista por defecto solo una vez al leer el archivo. Si la modificas en la primera llamada, ¡se mantiene modificada para todas las siguientes llamadas! La solución: def agregar(item, inv=None): if inv is None: inv = [].

4. Sobreescritura de Funciones Nativas ▼

❌ Lo que intentas:

sum = 100
# ...mucho código después...
total = sum([10, 20, 30])  # TypeError: 'int' object is not callable

✅ Lo que pasó:

Usaste la palabra `sum` (que es una función interna de Python) como nombre para una simple variable. Acabas de "destruir" la función original. Cuando más abajo intentas usarla como función `sum()`, Python dice "Oye, `sum` ahora es el número 100, no puedes llamar a un número con paréntesis". No uses palabras reservadas o nativas como nombres de variables (list, str, int, sum, max, min).

5. El Paso por Referencia "Silencioso" ▼

❌ Lo que intentas:

def borrar_malos(alumnos):
    alumnos.clear()
    return alumnos

mi_curso = ["Ana", "Juan", "Pedro"]
borrar_malos(mi_curso)
print(mi_curso) # []  ¡Me borró mi lista original!

✅ Lo que pasó:

Cuando pasas una Lista o Diccionario a una función, no estás pasando una copia, estás pasando el objeto original. Si la función le hace `.clear()`, `.append()`, o altera el diccionario, modifica tus datos originales. Si quieres evitar esto, pasa una copia: borrar_malos(mi_curso.copy()).

30 Retos Funcionales

Aplica todo lo aprendido. Codifica estas 30 funciones en tu IDE. ¡La dificultad aumenta progresivamente!

Nivel 1: Fundamentos (5 - 10 min c/u)

1. Saludo Personalizado

Crea saludar(nombre) que retorne el string "Hola, [nombre], bienvenido al sistema".

2. Calculadora Simple

Crea sumar(a, b) que retorne la suma de los dos números recibidos.

3. Número Par

Crea es_par(numero) que retorne True si el número es par, y False si es impar.

4. Mayor de Tres

Crea mayor_de_tres(a, b, c) que reciba 3 números y retorne el mayor de ellos.

5. Contador de Vocales

Crea contar_vocales(texto) que reciba un string y retorne la cantidad de vocales que contiene.

6. Reversa

Crea reversar_texto(texto) que devuelva el string al revés. (Ej: "hola" -> "aloh")

7. Calculadora de Propinas

Crea calcular_propina(total, porcentaje=10) que retorne el monto de la propina. El porcentaje por defecto es 10%.

8. Filtrar Pares

Crea filtrar_pares(lista_numeros) que reciba una lista, y retorne una NUEVA lista solo con los números pares.

9. Conversor de Temperaturas

Crea celsius_a_fahrenheit(c) que retorne la temperatura en F. Fórmula: (C * 9/5) + 32.

10. Generador de Password

Crea generar_password(longitud=8) que retorne un string aleatorio de la longitud especificada.

11. Diccionario de Frecuencias

Crea contar_caracteres(texto) que retorne un diccionario donde las claves son las letras y los valores son la cantidad de veces que aparecen.

12. Validador de Email

Crea es_email_valido(email) que retorne True solo si el texto contiene al menos un "@" y un "."

13. Promedio de Lista

Crea promedio(lista) que retorne el promedio de una lista de números. Retorna 0 si la lista está vacía.

14. Buscar en Diccionario

Crea buscar_alumno(rut, diccionario_alumnos) que retorne el nombre del alumno si el rut existe, o "No encontrado" si no.

15. Función Compuesta

Usando filtrar_pares() y sumar(), crea sumar_pares_lista(lista) que reciba una lista y retorne la suma de todos los números pares en ella.

Nivel 2: Misiones de Desarrollo (20 - 30 min c/u)

16. Analizador de Textos Modular

Crea analizar_texto(texto) que use internamente otras funciones para contar vocales, contar palabras y determinar la palabra más repetida. Debe retornar un diccionario con estos tres resultados.

17. Sistema de Combate RPG

Crea atacar(atacante, defensor) recibiendo diccionarios de Stats. Debe calcular evasión con random, reducir la vida del defensor mitigada por su armadura, y retornar una tupla (nueva_vida, log_combate).

18. Validador de Contraseñas Hardcore

Crea validar_password(psw) que retorne una tupla (True, "Éxito") o (False, "Razón del error"). Reglas: >8 caracteres, 1 mayúscula, 1 número, 1 símbolo especial (!@#$%^&*).

19. Cajero Automático (Greedy Algorithm)

Crea retirar_dinero(monto, saldo). Valida que haya saldo suficiente, resta el dinero, y retorna un diccionario con la cantidad exacta de billetes de 20k, 10k, 5k y 1k que la máquina debe entregar.

20. Gestor de Inventario Mutante

Crea funciones agregar_item(inv, item) y eliminar_item(inv, item) que modifiquen el diccionario de inventario original pasado por parámetro. ¡No uses returns, usa la referencia de memoria!

21. Buscaminas (Scanner de Matriz)

Crea contar_minas(matriz, fila, col) que reciba una matriz NxN donde "X" es mina. Debe retornar la cantidad de minas adyacentes a la coordenada (fila, col).

22. Cifrado César

Crea cifrar(texto, desp) y descifrar(texto, desp). Transforma el string sumando el desplazamiento al valor ASCII de cada letra. Preserva espacios.

23. Validador de Tarjetas (Luhn)

Investiga e implementa una función validar_luhn(numero_tarjeta) que determine si un número de tarjeta de crédito es matemáticamente válido usando el algoritmo de Luhn.

24. Compresor de Strings (RLE)

Crea comprimir_texto(texto). Si recibe "AAAABBBCCDAA", debe retornar "A4B3C2D1A2".

25. Calculadora Universitaria

Crea situacion_academica(ramos). Recibe una lista de diccionarios [{'nota': 5.0, 'pond': 30}, ...]. Debe validar que la ponderación suma 100%, calcular el promedio ponderado, y retornar si aprueba o va a examen.

26. Filtro Anti-Toxicidad

Crea censurar(texto, palabras_baneadas) que reemplace todas las incidencias de las malas palabras en el string por "***", pero preservando las mayúsculas/minúsculas originales del texto.

27. Máquina Vending Inteligente

Crea dar_vuelto(precio, pago). Si el pago es menor, retorna un string de error. Si es exacto, retorna diccionario vacío. Si es mayor, retorna diccionario con las monedas exactas de $500, $100, $50 y $10.

28. Batalla Naval (Disparo)

Crea disparar(tablero, x, y). Debe buscar en la matriz de la flota rival. Modifica el tablero original dejando un "X" si había barco (retorna "Impacto") o una "O" si era agua (retorna "Agua"). Retorna "Ya disparaste ahí" si repite.

29. Gestor de Fechas Bisiestas

Crea sumar_dias(fecha_str, dias). Debe recibir algo como "15/02/2026", sumar la cantidad de días respetando la cantidad de días por cada mes, y usando una función auxiliar es_bisiesto(year). Retorna la nueva fecha como string.

30. Tic-Tac-Toe AI Básica

Crea mejor_movimiento(tablero, simbolo_cpu). Debe analizar el tablero 3x3 actual, detectar si puede ganar en el próximo movimiento y retornar esa tupla (f, c). Si no, retornar la coordenada para bloquear al rival. Si no, retornar cualquier casilla libre.
