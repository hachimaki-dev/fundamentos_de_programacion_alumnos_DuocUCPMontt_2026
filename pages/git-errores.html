🚑 ¡Auxilio, Git! Clínica de Errores

Todos rompemos algo de vez en cuando. No entres en pánico, respira profundo. Aquí tienes el botiquín de primeros auxilios para salvar tu código. 🩹

📝 1. "Me equivoqué en el mensaje del commit"

Hiciste git commit -m "Solución de la gía" y te diste cuenta de que escribiste "gía" en vez de "guía" o querías agregar más detalles.

La Cura: Si aún no has hecho push, puedes modificar el mensaje del último commit fácilmente:

git commit --amend -m "Solución de la guía"

🗑️ 2. "Borré código importante y quiero volver a como estaba antes de modificarlo"

Estuviste programando, borraste una función que funcionaba perfecto, y ahora nada compila. Todavía no has hecho add ni commit.

La Cura: Puedes descartar todos los cambios no guardados en un archivo y dejarlo como estaba en tu último commit:

# Para restaurar un archivo específico
git restore nombre_del_archivo.py

# Para descartar TODO lo que no hayas guardado (¡Cuidado, esto no se puede deshacer!)
git restore .

⏪ 3. "Hice un commit, pero me arrepentí"

Hiciste commit, pero te faltó incluir un archivo o el código estaba roto y quieres "deshacer" ese commit sin perder tu trabajo actual.

La Cura: Usamos reset. Esto borra el commit del historial de Git, pero mantiene tus archivos intactos en tu carpeta para que puedas arreglarlos y hacer otro commit.

git reset --soft HEAD~1
⚠️ Peligro: Si usas git reset --hard HEAD~1, borrarás el commit y también borrarás los cambios en tus archivos. Úsalo solo si de verdad quieres destruir lo último que hiciste.

⚔️ 4. "¡Git dice que tengo Conflictos (Merge Conflicts)!"

Hiciste git pull y te salió un mensaje aterrador: "Automatic merge failed; fix conflicts and then commit the result."

Esto pasa cuando tú y un compañero (o tú en otro PC) editaron la misma línea de código y Git no sabe cuál versión es la correcta.

La Cura: Abre tu código en VS Code. Verás algo así:

<<<<<<< HEAD
print("Hola, Profesor Carlos")
=======
print("Hola, Mundo")
>>>>>>> rama-del-companero

Sigue estos pasos:

  1. Abre el archivo con conflicto.
  2. Borra las líneas raras de Git (<<<<<<<, =======, >>>>>>>).
  3. Deja el código exactamente como quieres que quede finalmente.
  4. Guarda el archivo.
  5. Haz git add . y luego git commit -m "Resolviendo conflictos". ¡Listo!

🔑 5. "GitHub me dice 'Permission denied' al hacer Push"

Intentas hacer git push y te sale un error de autenticación. Probablemente GitHub no reconoce quién eres.

La Cura: Asegúrate de que hayas iniciado sesión en VS Code o en tu terminal. Si Git te pide configurar tu email y nombre, ejecuta:

git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"
