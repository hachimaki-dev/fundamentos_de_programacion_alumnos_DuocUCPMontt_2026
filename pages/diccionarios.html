📚 Diccionarios en Python

¡Bienvenid@ al mundo de los pares clave:valor! 🔑 Los diccionarios son la estructura más poderosa para organizar datos con etiquetas descriptivas en vez de índices numéricos.

🧐 El Problema: Datos Sin Contexto

Con las listas aprendiste a guardar muchos datos en una sola variable. Pero... ¿y si necesitas saber a quién pertenece cada dato? Imagina tener ["Ash", 10, "Pueblo Paleta"]. ¿El 10 es edad? ¿Nivel? ¿Pokédex? Sin contexto, tus datos son un rompecabezas sin imagen de referencia.

La solución: "Necesito almacenar datos CON etiquetas descriptivas que me digan exactamente QUÉ representa cada valor".

📚 El Diccionario: Tu Base de Datos Personal

Un diccionario es una colección de pares clave:valor. Piensa en él como un directorio telefónico: buscas por nombre (clave) y obtienes el número (valor). No necesitas recordar posiciones numéricas.

🔑 La Diferencia Clave vs Índice

En las listas accedes por posición numérica: lista[0]. En los diccionarios accedes por etiqueta descriptiva: diccionario["nombre"]. Es como la diferencia entre buscar "el cajón #3" vs buscar "el cajón que dice CALCETINES".

🔍Anatomía de un Diccionario
# Creación con llaves { } y pares clave: valor
personaje = {
    "nombre": "Ash",
    "edad": 10,
    "region": "Kanto",
    "pokemones": ["Pikachu", "Charizard"]
}

# Acceso por CLAVE (no por índice numérico)
print(personaje["nombre"])   # Ash
print(personaje["edad"])     # 10

# Agregar un par nuevo es directo
personaje["medallas"] = 8

# Modificar un valor existente
personaje["edad"] = 11

☢️ El Horror del KeyError

Si intentas acceder a una clave que NO existe usando corchetes — personaje["novia"] — Python colapsa con un 🚨 KeyError. Solución segura: usa .get("novia", "No tiene") que devuelve un valor por defecto sin romper nada.

🛠️ Métodos que DEBES Dominar

Un diccionario sin métodos es como un cofre del tesoro sin llave. Memoriza estas operaciones y domina Python:

dict['clave'] = valor

Agregar o modificar: Si la clave no existe, la crea. Si ya existe, sobrescribe el valor anterior sin avisar.

pokedex["Bulbasaur"] = "Planta" # Agrega nuevo par

.get(clave, default)

Acceso seguro: Busca la clave sin riesgo de KeyError. Si no existe, devuelve el valor por defecto que tú elijas.

nivel = jugador.get("nivel", 1) # Si no tiene nivel, devuelve 1

.keys() / .values() / .items()

Los 3 mosqueteros: .keys() devuelve solo las claves, .values() solo los valores, y .items() devuelve pares (clave, valor) — perfecto para recorrer con for.

for clave, valor in diccionario.items(): print(clave, valor)

del dict['clave']  |  .pop('clave')

Eliminar pares: del borra directamente. .pop("clave") borra y además te devuelve el valor eliminado.

del persona["edad"] # Elimina el par edad:valor
eliminado = persona.pop("region", "N/A") # Borra y guarda

⚔️ Lista vs Diccionario: ¿Cuándo Uso Cada Uno?

Esta es la pregunta del millón. La respuesta depende de UNA cosa: ¿necesitas etiquetas o solo posiciones?

📦

LISTA: Colección Ordenada

  • 🔢 Acceso por posición numérica: lista[0], lista[1]...
  • 📋 Ideal para colecciones homogéneas: notas, nombres, precios.
  • 🎯 Úsala cuando: solo necesitas almacenar muchos datos del mismo tipo y recorrerlos en orden.
📚

DICCIONARIO: Pares con Nombre

  • 🔑 Acceso por clave descriptiva: dict["nombre"], dict["edad"].
  • 🗂️ Ideal para datos heterogéneos con contexto: ficha de estudiante, configuración, perfil.
  • 🎯 Úsalo cuando: cada dato necesita una etiqueta que explique qué representa.

🚀 Recorriendo Diccionarios: Los 3 Modos

El ciclo for sobre diccionarios tiene 3 modos según lo que necesites extraer: solo claves, solo valores, o ambos.

1 MODO: Solo CLAVES (por defecto)

Por defecto, iterar sobre un diccionario recorre solo las claves. Útil cuando necesitas verificar existencia o acceder a valores manualmente.

🐍Recorrido por Claves
pokedex = {"Pikachu": "Eléctrico", "Charmander": "Fuego", "Squirtle": "Agua"}

# Por defecto, el for recorre las CLAVES
for pokemon in pokedex:
    print(pokemon)
# Imprime: Pikachu, Charmander, Squirtle

# Equivalente a: for pokemon in pokedex.keys():

2 MODO: CLAVE + VALOR con .items()

El modo más poderoso. Con .items() desempaquetas clave y valor simultáneamente en dos variables. Es el patrón estrella para procesar diccionarios.

🐍Desempaquetado con .items()
pokedex = {"Pikachu": "Eléctrico", "Charmander": "Fuego"}

# .items() entrega tuplas (clave, valor) - usamos DOS variables
for pokemon, tipo in pokedex.items():
    print(f"{pokemon} es de tipo {tipo}")
# Imprime: Pikachu es de tipo Eléctrico
#          Charmander es de tipo Fuego

Aplicación Práctica y Retos

Pon tu IDE en marcha e implementa estas soluciones maestras.

🟢 Reto Fácil: Pokédex Básica

Ash quiere registrar a sus primeros 3 Pokémon con su tipo elemental en la Pokédex.

Instrucciones:

  • Crea un diccionario pokedex = {"Pikachu": "Eléctrico", "Charmander": "Fuego", "Squirtle": "Agua"}.
  • Usa un ciclo for pokemon, tipo in pokedex.items(): para iterar.
  • Dentro del ciclo, imprime en consola "El Pokémon [nombre] es de tipo [tipo]".
  • Al salir, usa len() para imprimir "Ash tiene [X] Pokémon registrados".

🟡 Reto Medio: Registro de Alumnos

El profesor necesita calcular el promedio de notas de cada alumno a partir de un diccionario que contiene listas de notas.

Instrucciones:

  • Crea el diccionario base: alumnos = {"Ana": [6.5, 7.0, 5.5], "Luis": [4.0, 3.5, 5.0], "Marta": [7.0, 6.8, 6.5]}.
  • Recorre el diccionario con for nombre, notas in alumnos.items():.
  • Dentro del ciclo, calcula el promedio sumando las notas y dividiendo por la cantidad: prom = sum(notas) / len(notas).
  • Evalúa si el promedio es mayor o igual a 4.0. Imprime "[Nombre] tiene promedio [Prom] - APROBADO" o "REPROBADO".

🔴 Reto Avanzado: Sistema de Inventario RPG

Crea un sistema de inventario para un RPG donde el jugador pueda agregar, ver, modificar y eliminar objetos usando un menú interactivo.

Instrucciones:

  • Inicializa inventario = {} (diccionario vacío).
  • Crea un while True: con un menú numérico: 1-Agregar objeto, 2-Ver inventario, 3-Modificar cantidad, 4-Eliminar objeto, 5-Salir.
  • Si es 1: Solicita al usuario el nombre del objeto y su cantidad. Guárdalo con inventario[nombre] = cantidad.
  • Si es 2: Recorre el inventario con .items() y muestra todo en pantalla.
  • Si es 4: Solicita el nombre del objeto, verifica if nombre in inventario: y elimínalo usando la instrucción del o .pop().
