02
⚙️ Métodos de Ordenamiento

Métodos Nativos de Python

🔹

.sort()

El método .sort() ordena una lista directamente, modificando su contenido original sin crear una copia. 

numeros = [3, 1, 4, 2]
numeros.sort()
print(numeros)  # [1, 2, 3, 4]
// Presiona ▶ Ejecutar para ver la salida
✓ Modifica la lista original
✗ No retorna un nuevo valor
✓ Es eficiente
🔹

sorted()

La función sorted() crea una nueva lista ordenada sin modificar la original. 

numeros = [3, 1, 4, 2]
ordenados = sorted(numeros)
print(numeros)    # [3, 1, 4, 2]
print(ordenados)  # [1, 2, 3, 4]
// Presiona ▶ Ejecutar para ver la salida
✗ No modifica la lista original
✓ Retorna una nueva lista
✓ Muy útil para conservar datos

🔄 Bubble Sort

Algoritmo Paso a Paso

Bubble Sort compara elementos consecutivos y los intercambia si están en el orden incorrecto, haciendo que el mayor "burbujee" al final.

VISUALIZADOR INTERACTIVO

Presiona ▶ Visualizar para comenzar
Comparando Intercambiando Ordenado

💻 Implementación con Visualización en Consola:

def bubble_sort_visual(lista):
    n = len(lista)
    paso = 1
    
    print("Iniciando Bubble Sort...\n")
    
    for i in range(n):
        for j in range(0, n - i - 1):
            
            print(f"Paso {paso}: {lista}", end=" ")
            
            if lista[j] > lista[j + 1]:
                print(f"-> Intercambiando {lista[j]} y {lista[j + 1]}")
                lista[j], lista[j + 1] = lista[j + 1], lista[j]
            else:
                print(f"-> {lista[j]} y {lista[j + 1]} en orden")
            
            paso += 1
        
        print(f"Resultado parcial: {lista}\n")
    print("Lista ordenada:", lista)

numeros = [3, 1, 4, 2]
bubble_sort_visual(numeros)
// Presiona ▶ Ejecutar para ver la salida
05
🔍 ¿Cómo Funciona?

Mecanismo de Bubble Sort

1️⃣

Recorre la lista

Se hacen múltiples pasadas sobre todos los elementos de la lista.

2️⃣

Compara vecinos

En cada paso compara dos elementos consecutivos entre sí.

3️⃣

Intercambia si hay desorden

Si el izquierdo es mayor que el derecho, los intercambia.

4️⃣

El mayor "sube"

Al final de cada pasada, el valor mayor queda al final como una burbuja que sube.

⏱️ Complejidad Algorítmica

Eficiencia Comparada

🐢

Bubble Sort

O(n²)
  • ▸ Lento para listas grandes
  • ▸ Cuadrático en el peor caso
  • ▸ Útil para aprender conceptos
⚡

.sort() / sorted()

O(n log n)
  • ▸ Mucho más eficientes
  • ▸ Usan Timsort
  • ▸ Recomendados en producción

COMPARACIÓN VISUAL DE EFICIENCIA (n = 100 elementos)

Bubble Sort O(n²) ~10,000 ops
Timsort O(n log n) ~664 ops
07
🎯 Concepto Clave

In-Place vs Nueva Lista

IN-PLACE

.sort()

Modifica la lista original en memoria. No crea una copia. Ideal cuando no necesitas el estado anterior.

lista → [ordenada]
// misma referencia

NUEVA LISTA

sorted()

Crea una copia ordenada manteniendo el original intacto. Ideal para pipelines de datos o análisis.

lista → [original]
nueva → [ordenada]
08
🧾 Conclusión

¿Cuándo usar cada método?

⚡

Usa .sort()

Si quieres eficiencia y no necesitas conservar la lista original. Opera directamente en memoria.

🛡️

Usa sorted()

Si necesitas mantener los datos originales intactos. Perfecto para análisis y transformaciones de datos.

📚

Bubble Sort

Permite entender la lógica del ordenamiento. Excelente para aprender, pero no eficiente para producción con listas grandes.

Dominar los algoritmos de ordenamiento es el primer paso para comprender cómo las computadoras organizan y procesan información de forma eficiente.
